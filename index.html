Immutability

Immutability is an important concept that should be thoroughly understood by JavaScript programmers

What is Immutability?

Before learning the meaning of Immutability in JavaScript, it would be reasonable to understand the meaning of Immutability first.

The dictionary can help us to define this word:

Something that is not subject to be changed

Example: Weather can go from sunny, to rainy and windy, and back to sunny again

Having this definition behind us, we can say that Immutability is the exact opposite of this

We should then expect that something Immutable would not be subject to change, no matter the circumstances.

What is Immutability refering to?

Immutability refers to primitive values

Primitive values cannot have anything added upon then, they can only be reassigned

Because of this, all primtive types in JavaScript are immutable

Primitive Types

  • string
  • number
  • boolean
  • undefined
  • null
  • symbol

Reference Types

  • objects
  • arrays
  • functions
